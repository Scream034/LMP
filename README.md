
# 🎵 MyLiteMusicPlayer 

**Lite Music Player** — это производительный, легковесный и современный десктопный клиент для YouTube Music. Проект создан для тех, кто ценит скорость работы, минималистичный интерфейс и качественный звук без лишней нагрузки на систему.

## ✨ Основные особенности

*   🚀 **Экстремальная производительность:** Написан на .NET 10 с использованием Avalonia UI. Весь интерфейс оптимизирован для плавной работы даже на слабых ПК.
*   🎧 **Умный стриминг (Memory-First):** Собственная реализация `MemoryFirstCachingStream` с механизмом Read-ahead. Трек начинает играть мгновенно, кешируясь в оперативную память и параллельно сохраняясь на диск.
*   🖼️ **Spotify-inspired UI:** Эстетичный темный интерфейс с использованием "скелетной" анимации загрузки (Skeletons), плавными переходами и кастомными контролами.
*   📦 **Кеширование всего:** Изображения, результаты поиска и аудиопотоки кешируются локально, что минимизирует количество запросов к сети и экономит трафик.
*   🔊 **Продвинутый звук:** Движок на базе LibVLC с поддержкой усиления (Gain Control), лимита громкости и плавного переключения треков.
*   🌍 **Локализация:** Полная поддержка русского и английского языков (i18n).
*   📚 **Личная библиотека:** Управление плейлистами, система лайков и история прослушиваний, работающая в офлайн-режиме.

## 🛠 Технологический стек

*   **UI Framework:** [Avalonia UI](https://avaloniaui.net/) (MVVM, ReactiveUI).
*   **Audio Engine:** [LibVLCSharp](https://github.com/videolan/libvlcsharp) (VLC под капотом).
*   **YouTube Integration:** [YoutubeExplode](https://github.com/Tyrrrz/YoutubeExplode).
*   **Image Loading:** AsyncImageLoader с кастомным дисковым кешем.
*   **Dependency Injection:** Microsoft.Extensions.DependencyInjection.

## ⚙️ Технические "фишки"

### Memory-First Caching Stream v7
В отличие от обычных плееров, MyLiteMusicPlayer использует кастомный поток для аудио:
1.  **Chunked requests:** Загрузка большими блоками для обхода ограничений скорости YouTube.
2.  **Adaptive Buffer:** Автоматическая пауза загрузки, если буфер заполнен, и возобновление при необходимости.
3.  **Disk Persistence:** Если вы прослушали трек один раз, при следующем запуске он будет взят из локального кеша.

### UI Оптимизация
*   Использование `InfiniteScrollBehavior` для бесконечных списков без лагов.
*   LRU-кеш для изображений (память + диск).
*   Троттлинг API-запросов для предотвращения блокировок со стороны сервисов.

## 🚀 Запуск

**Требования:**
*   .NET 10 SDK (проект использует самые свежие возможности платформы).
*   Установленный в системе или поставляемый с приложением VLC.

```bash
# Клонировать репозиторий
git clone https://github.com/Scream034/MyLiteMusicPlayer.git

# Перейти в папку
cd MyLiteMusicPlayer

# Запустить проект
dotnet run
```
